Tech Stack Options

Three approaches for your portfolio — all targeting Cloudflare Pages with a headless CMS and inline book reader.

A

Astro + Sanity CMS Recommended

Best fit for a portfolio. Astro generates ultra-fast static HTML by default and has first-class Cloudflare Pages support. You can drop in React components (like a PDF viewer) as "islands" — so your React/Next.js knowledge still applies. Sanity gives you a polished CMS dashboard to swap the book file anytime.

Pros

  • Fastest possible output — pure HTML where JS isn't needed
  • Cloudflare Pages adapter is rock-solid
  • React islands = familiar component model
  • Sanity free tier is generous

Cons

  • New framework to learn (though easy to pick up)
  • Less ecosystem than Next.js
B

Next.js (static export) + Sanity CMS

Familiar territory. You already use Next.js in your Neo4j project. A static export (next export) runs perfectly on Cloudflare Pages with no server needed. Sanity handles the book. The trade-off is that static export disables Next.js server features (API routes, SSR) — fine for a portfolio.

Pros

  • You already know Next.js + React + TypeScript
  • Huge ecosystem, tons of examples
  • Works well on Cloudflare Pages via static export

Cons

  • Static export loses API routes & SSR
  • Heavier JS bundle than Astro for a mostly-static site
  • More boilerplate for a simple portfolio
C

Next.js + Cloudflare Workers adapter + Sanity CMS

Full SSR on Cloudflare. Using @cloudflare/next-on-pages you get server-side rendering at the edge — API routes work, dynamic content works. But the adapter has limitations (not all Next.js features are supported) and adds real setup complexity.

Pros

  • Full Next.js capabilities including API routes
  • Edge SSR — fast globally
  • Future-proof for dynamic features

Cons

  • Complex setup — adapter has known gotchas
  • Not all Next.js features supported on Cloudflare Workers
  • Overkill for a portfolio + book reader

All three options use Sanity CMS (free tier) for book management and react-pdf / PDF.js for the inline reader. Click your preferred option above, or tell me in the terminal.
